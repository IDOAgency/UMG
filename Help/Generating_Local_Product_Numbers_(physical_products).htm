Generating Local Product Numbers (physical products)

Local Product Numbers are required before you can save any physical product. There are two methods for doing this:

·        You can auto-generate Local Product Numbers using the Generate Local Product # button.

·        You can enter Local Product Numbers manually. For commercial products, simply enter the Local Product Number from the block assigned to your company or label. For promotional product, use the pulldown to select the appropriate promo prefix, then enter Local Product Number from the block assigned to your company or label.

Procedures for both methods are given below.

Keep in mind…

In addition to the fields (described above) required to save Selection screens, others are required to auto-generate local product numbers. In this section, these fields are discussed along with procedures for entering them. The discussion then turns to the auto-generate function button.

Entering Config Codes

Follow these steps:

1.      Pull down the Config Code list.

2.      Choose the appropriate Legacy Configuration Code.

3.      Move cursor to UPC field.

4.      Enter UPC. Just enter the digits, do not use spaces or hyphens. Example: 0731453412345

Entering SoundScan Group Codes

The SoundScan Group code links different versions of essentially the same release for the purpose of SoundScan reporting. That is, you can link the explicit and edited versions of an album, as well as any generic, value-added versions which might be applicable. Follow these steps to enter SoundScan Group code.

5.      Move your cursor to the SoundScan Grp text box.

6.      Enter the UPC of the first version being released.

Keep in mind…

·        If multiple versions are being released simultaneously, use the UPC of the regular (non-edited and non-value-added) product.

·        If a limited edition or value-added version is being released before the regular version, use the UPC of the first version being released.

·        If you’re doing a value-added release for a specific account (such as Best Buy), be sure to have your label sales contact notify UMVD so that it can be manually linked for SoundScan.

·        For commercial singles, follow the same guidelines given here for full-length. Make sure track lists are the same for all linked singles (CD singles links to cassette singles and so on). Use the UPC of the first single version being released as SoundScan Group code. You cannot link different formats (CD with DVD, for example) for SoundScan, even if the title is identical.

Clicking the button - messages

Once all required fields have been entered, you’re ready to auto-generate the local product number. There are several messages which might appear when you click the button, depending on your entries in the Selection field up to this point. All are discussed below.

7.      Click the Generate Local Product # button. Depending on your entries in Selection screen fields, you might get one of the following messages:

·        If any required field is blank, a message is displayed listing the required field(s) that are blank. Complete those fields, then click the Generate Local Product # button again.

·        If the Title ID is blank, a prompt (shown) is displayed, asking if you want Title ID to match the Local Product Number you’re generating. Do one of the following:

·        Click OK, and the local product number is entered into the Title ID field.

·        Click Cancel and a prompt reminds you to enter a Title ID. Click OK to continue.

Keep in mind…

When local product number is generated and the Config Code indicates the product is a single, the Local Product Number is entered automatically into the Title ID field as Legacy does not permit linking singles using Title ID.

Transferring Local Product Numbers to Legacy

Clicking the Generate Local Product # button sets in motion a number of operations involving both Milestone and Legacy systems. When the button is clicked the first time, required fields from the Selection screen are transferred to Legacy, and an entry is created in Legacy for your UML Production contact. This transmission occurs only once, the first time you generate a Local Product Number. Subsequent changes to the Local Product Number are not transmitted to Legacy.

Legacy actually generates the Local Product Number, then sends it back to Milestone. To do this correctly, Legacy looks at two key fields:

·        Title ID is used to group different formats of identical releases for internal reporting. That is, using this field you can link CDs, cassettes and LPs of the exact same releases. You cannot, however, link explicit CDs to edited CDs as their track lists are not identical. Title ID is used by Legacy to assign the base product number. Example: B1234567.

·        Config Code is used by Legacy to append a config digit (such as 02 for compact discs) to the base product number. Example of complete Local Product Number: B1234567-02.
