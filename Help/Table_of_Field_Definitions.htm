Table of Field Definitions

 

 

Field Name

P: physical only

D: digital only

B: Both

 

 

Description

B Side

P

Text box containing second side, or track, of single release.

Bundle Title

D

Required. Text box containing title of tracks being released.

Company

B

Read-only field, imported with Archimedes project information, containing the company name assigned to artist roster entry in Archimedes (Geffen, A&M, Mercury, GRP and so on).

Config Cd

B

Required to auto-generate local product number. Pulldown list used to specify Legacy Configuration Code (CD, DVD-Video, Cassette and so on). This code also drives the check digits at the end of Local Product Numbers.

Digital Release Date

B

Required for digital product. Date field containing date of release (to the public) for the digital product.

Division

B

Read-only field, imported from Archimedes project information, which contains financial entity name (Blue Gorilla, Tuff Gong, Silas, Blue Thumb and so on).

Environment

B

Read-only field, imported with Archimedes project information.

First Name

B

Text box containing artist’s first name.

Format

P

Required. Pulldown list used to specify format (Vinyl, Compact Disc, Cassette, DVD-Video and so on) of physical product.

GRid#

D

Field containing unique numbers which identify releases of sound recordings for electronic distribution.  

Hold

P

Check box used to indicate product release is temporarily being held.

Hold Reason

P

Popup window used to enter reasons for hold.

Imprint

B

Required. Text box used to contain the label logo associated with artist. In most cases it is the same as the Corporate Financial Label imported from Archimedes. 

Int’l Date

P

Date field containing date of international release for physical product.

Int’l Only

P

Checkbox used to indicate product is only for international release, for which Milestone is used only to track international deadlines.

Label

B

Read-only field, imported with Archimedes project information, used to indicate the financial entity name  at the lowest level (Republic, Booga Basement, Magic and so on).

Label Contact

B

Pulldown list used to specify the name of the label production person responsible for the product. Default entry is name of user creating product record, but can be changed.

Last / Group Name

B

Required. Text box containing last name of the individual artist or full name of artist group.

Last Street Date Update/Last Release Date Update

B

Read-only field indicating date of last change to Street/Ship Date or Digital Release Date field.

Last Updated

B

Read-only field indicating name of user who last saved product, as well as time and date of save.

Local Product #

P

Required. Text field used to contain unique identifying number for the product using format, Bnnnnnnn-nn. Can be entered manually for promotional products or for products from labels not set up for automatic product number generation (APNG). For labels that are setup for APNG this number can be generated automatically through Legacy.

Music Type

B

Pulldown list used to specify music genre as well as to determine UMVD sales reps responsible for product marketing.

New Bundle

 

Exact Duplicate of Physical Product

D

Required. Radio buttons indicating:

 

New Bundle – no physical product equivalent exists for the track(s) on digital product.

 

Exact Duplicate of Physical Product – digital product is an exact duplicate of an existing physical product.

Oper Company

B

Read-only field, imported with Archimedes project information, which contains the two-digit alphanumeric Legacy code uniquely linked to corporate structure, and which identifies the treatment of royalties for the product. 

Origin Date

B

Read-only field containing the date record was created in Milestone.

Other (Contact)

B

Text box used to hold name of additional information for product, such as name of salesperson, A&R representative or track list. This field also can be used to customize Milestone reports.

P&D / Dist.

B

Checkbox, populated based on Archimedes project information, used to indicate contractual relationship between releasing label and UMG, either press and distribute (P&D) or distribution only (Dist.).

Parental Advisory

B

Checkbox indicating special labeling for parental advisory is required due to explicit lyrics.

Physical Prefix /Local Prod #

D

Information only field which displays the product number of the corresponding physical release.

Physical Street Date

D

Date field containing date of release (to public), or street date, for corresponding physical release.

Pkg

P

Popup window containing text box used to specify packaging details.

Prefix

P

Pulldown list used to contain prefix for promotional products. This field is filtered by environment and manually selected.

Price

B

Read-only field indicating selling price, auto-populated based on price code entries, and used for documentation purposes only.

Priority

D

Checkbox used to indicate priority processing is required to ensure the release date for the product is met. Indicates that Digital Release Date is a hard date.

Prod Category

B

Required. Pulldown list used to specify classification (box set, catalog, new release or soundtrack and so on) of the product being released and to drive suggested Schedules in Milestone.

Product Title

P

Required. Text box used to enter product title.

Project ID

B

Read-only field, imported with Archimedes project information, a unique identifier for the associated project.

Radio Impact Date

B

Hyperlink-triggered popup window used to indicate radio impact date for various radio broadcast formats.

Release Type

B

Required. Pulldown list used to specify release as either commercial or promotional.

Releasing Family

B

Pulldown list used to specify marketing label group (IGA, IDJ and so on).  Default entry is taken from User Info screen.

 

Note: Releasing Family is not always the same as the Label Family.

Retail

B

Pulldown list used to specify retail price code, which is based upon Sell price code.

Schedule Type

D

Required. Pulldown list (Artist Session, Digital Only, Digital Prior to Physical, Digital Simultaneous w/Physical) for which entries describe whether product is associated with physical release.

Selection Comments

B

Popup window, triggered by hyperlink, used to enter supplemental information about product. This field also is accessible from Schedule screen.

Sell

B

Dropdown list used to specify valid price code, for documentation purposes only.

Sell Filter

B

The sell code filter.

SoundScan Group Code

P

Required to generate Local Product Numbers. Text box used to specify an identifier linking separate products of essentially the same album for SoundScan reporting purposes.  These Products do not have to be exactly the same in order to be linked.  For all products in a group the SoundScan Group Code should be the UPC from the original/regular version (usually the explicit version of the CD, if there is one). 

Spec Pkg

P

Checkbox used to indicate non-standard packaging (box set, digipak and so on) is required for the product. Informs accounts product might not be compatible with automated inventory machines.

Special Instructions

D

Text box used to enter special instructions for the eCommerce Department.  Once entered here, text appears in the Digital Production Schedule Report.

Status

B

Pulldown list used to indicate status of the product (Active, TBS, Cancelled or Closed).

Street/Ship Date

P

Required. Date field used to contain street date for commercial product, ship date for promotional product.

Sub Label

B

Read-only field, imported with Archimedes project information, containing Legacy code identifying releasing label.

Sub-Format

B

Required. Pulldown list used as second level of classification for format (full length, single, maxi and so on).

Super Label

B

Read-only field, imported with Archimedes project information, which contains Legacy identifying code for the releasing label at the lowest level.

Territory

B

Text box used to specify territories into which product is being released.

Title ID

P

Required. Text box used to enter the Local Product Number to which product should be linked. Title ID links different formats of the exact same track list together for internal UMG reporting purposes.

UPC

B

Required to generate Local Product Numbers. Text box used to enter Universal Product Code for commercial Products.
