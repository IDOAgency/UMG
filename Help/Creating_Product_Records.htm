Creating Selections

Follow these steps to create new product records:

1.      Click Selection in the menu bar under the Labels tab. (Typically, the Labels tab is activated by default when you log on.) The Selections screen appears.

2.      If the New button does not appear, you do not have edit access to the product you’ve selected. In this case, search for or navigate to a product for which you have edit rights. The New button should appear. See Basic Training for procedures for locating products.

3.      Click the New button (at the top of the Selection screen). The Project Search window appears. See Searching for Projects, below, to continue.

Searching for projects: the Project Search Window

Search  results for Benoit in the Last/Group Name field. Notice the unknown project  for the Verve label.

 

4.      Click either Physical or Digital radio button (at top) depending on the type of release you wish to enter.

5.      Enter search criteria using First Name, Last/Group Name, Project Title, Project Description or Project ID. You can enter partial search targets, if needed. Use the Label pulldown to search by Releasing Label.

6.      Click Search or press Enter to execute your search. Results are displayed in the list window at bottom. You can sort your results by clicking on column headings in blue. The blue triangle indicates which column is used as sort key and the sequence (ascending/descending) of the sort. Default sort is by Create Date, descending. That is, the most recently projects appear at top.

Note

For overall system performance, Milestone limits your project list to 200 projects. If your search returns more than 200 projects, a message appears at the top of the search window stating that only the first 200 are listed. If you see this message, enter more specific search criteria to limit your results to a more manageable number.

Or do one of the following:

·        Click Cancel to return to the Selection screen without specifying a project.

·        Click Clear to remove all entries and re-enter your search criteria.

7.      Double click the project you wish to use. The Selection screen – either physical or digital depending on your entries – returns. Project identifying data populates the appropriate fields. Notice some fields (Artist Name, Title and Imprint) can be overwritten; others are read-only.

Hint

An Unknown Project appears for every label family to which you have access. Use these unknown projects as placeholders for projects which haven’t yet been set up in Archimedes. Then use the Revise button in the Selection screen later, to import the appropriate project data – and overwrite the Unknown entries in Artist, Title, Description and Label fields –  when it becomes available.

Searching for UMVD projects (UML production users only)

UML production users must follow a slightly different procedure to locate projects for their products. Follow these steps:

·     Click the New button (at the top of the Selection screen). The Project Search window appears. By default, the Physical radio button is checked.

·     Enter your search criteria, using either the Label pulldown list or the Project ID text field.

·     Click Search or press Enter. Results appear at bottom.

·     Double click on the project you wish to use. Selection screen returns, project identifying fields are entered automatically.

NOTE – UML production users only

When searching by Project ID, omit the leading zero and the hyphen. Example: for Project ID 0069-1235, enter 0691235 in the Project ID field in the Project Search window.

Entering required fields

Fields with red asterisks next to them (an example is shown at left) must be entered before you can save the Selection screen. This section presents guidelines for completing these fields.

8.      Do one of the following:

·        For physical products, move cursor to Street/Ship Date field. Enter street date for commercial products, Ship date for promotional products.

·        For digital products, move cursor to Digital Release Date field. Enter date of digital release for the product.

·        Skip this step if the Status field is set to TBS (to be scheduled).

9.      Pull down the Status list and choose the appropriate status, if needed. Skip this step to use the default entry, Active. Choose TBS (to be scheduled) for product waiting for a date.

10.  Pull down the Releasing Family list and choose the appropriate entry if needed. Skip this step to use the default entry, which is the marketing label group assigned to your user ID.

11.  Move cursor to Imprint text box and overwrite as needed. Default entry is the label logo associated with the artist in Archimedes.

12.  Pull down the Product Category list. Choose the appropriate entry.

13.  Pull down the Release Type list. Choose Commercial or Promotional, depending on your product.

14.  Do one of the following:

·        For physical products, pull down the Format list and choose the appropriate entry.

·        For digital products, pull down the Schedule Type list and choose the appropriate entry.

15.  Pull down the Sub-Format list. Choose the appropriate entry.

NOTE:

·        At this point, digital products can be saved. To do this, simply click the Save button (there’s one at the top and bottom).

·        When creating physical products, complete as many remaining fields as needed for your product, then either a temporary product number or Local Product Number is required before you can save. See Generating Temporary Product Numbers and/or Generating Local Product Numbers in sections that follow.

Entering additional Comments and Info

You are given several choices for entering additional or supplemental information and comments in the Selection screen, including:

·        a general Comments field (physical/digital)

·        Special Instructions field (digital only)

·        the Pkg field features a Comments window popup (physical only)

All fields are optional. Procedures for making entries in these fields are given below.

Entering general comments (digital and physical)

A general Comments field, located next to the Street/Ship date field, can be used to enter supplemental information and is accessible from both Selection and Schedule screens. Follow these steps:

1.      Click the blank paper icon. The comments text box (shown) appears.

2.      Enter comments as needed.

3.      Click Save Comments to save. Click Cancel to return without saving. The icon changes to include a pencil when comments have been saved.

Entering special instructions (digital only)

Follow these steps to make entries in the Special Instructions field (above) in the digital Selection screen.

1.      Move cursor to Special Instructions text box.

2.      Enter text as needed.

Adding comments to the Pkg field (physical only)

Follow these steps to enter packaging-related comments in the physical Selection screen.

1.      Double click in the Pkg text box. Packaging popup window appears (shown). Releasing Family pulldown shows – this is a bug and that be fixed, says Tommy Thomas.

2.      Enter comments in text box as needed.

3.      Do one of the following:

·        Click Save Comments to retain comments and return to Selection screen.

·        Click Cancel to return to the Selection screen without retaining comments.

 

Entering sets in Milestone (optional)

Follow these steps to set up multiple unit sets, as in box sets or two-LP sets, in the Selection screen.

1.      Create and save a Selection screen for your set carrier, using either an auto-generated local product number or by entering a promo number. See the procedures above for details.

2.      Click on the UPC or Local Product # links. The component popup window appears (shown) when you click either link.

3.      Click Add. A blank line appears.

4.      Enter component’s Local Product Number.

5.      Enter component’s UPC in the text box provided.

6.      Enter a brief description of the component in the text box provided.

7.      Do one of the following:

·        Click the Add button again to enter another component. Repeat these steps until all components have been entered.

·        Click the Save button to save the component information you’ve entered. The Selection screen returns and both links (UPC and Local Product #) change to a purple color to indicate component information has been entered.

·        Click the del link to the left of any component you wish to remove.

8.      Click the Save button in the Selection screen to save the Selection record.

IMPORTANT

Even though you’ve clicked Save in the popup window, your component information is lost unless you save the Selection record. Thus, after entering and saving component information in the popup window, be sure to click the Save button from the Selection screen.

Designating projects as held (optional)

1.      Click the Hold checkbox to indicate your product is on hold. A checkmark (Ö) appears when hold has been activated.

2.      Use the Hold Reason comments field, next to the checkbox, to enter your reasons for putting the product on hold. Click the blank paper icon and the Hold Reason comments box (shown) appears.

3.      Enter text giving reason(s) for hold.

4.      Do one of the following:

·        Click Save Reason. Selection screen returns and a pencil now appears with the icon.

·        Or click Cancel to return without saving.

NOTE

If you enter text into this Hold Reason comments box before you click the Hold checkbox, the box is checked automatically when you click Save Reason and return to the Selection screen.

Entering Config Codes

Follow these steps:

1.      Pull down the Config Code list.

2.      Choose the appropriate Legacy Configuration Code.

3.      Move cursor to UPC field.

4.      Enter UPC. Just enter the digits, do not use spaces or hyphens. Example: 0731453412345

Entering SoundScan Group Codes

The SoundScan Group code links different versions of essentially the same release for the purpose of SoundScan reporting. That is, you can link the explicit and edited versions of an album, as well as any generic, value-added versions which might be applicable. Follow these steps to enter SoundScan Group code.

5.      Move your cursor to the SoundScan Grp text box.

6.      Enter the UPC of the first version being released.

Keep in mind…

·        If multiple versions are being released simultaneously, use the UPC of the regular (non-edited and non-value-added) product.

·        If a limited edition or value-added version is being released before the regular version, use the UPC of the first version being released.

·        If you’re doing a value-added release for a specific account (such as Best Buy), be sure to have your label sales contact notify UMVD so that it can be manually linked for SoundScan.

·        For commercial singles, follow the same guidelines given here for full-length. Make sure track lists are the same for all linked singles (CD singles links to cassette singles and so on). Use the UPC of the first single version being released as SoundScan Group code. You cannot link different formats (CD with DVD, for example) for SoundScan, even if the title is identical.

Clicking the button - messages

Once all required fields have been entered, you’re ready to auto-generate the local product number. There are several messages which might appear when you click the button, depending on your entries in the Selection field up to this point. All are discussed below.

7.      Click the Generate Local Product # button. Depending on your entries in Selection screen fields, you might get one of the following messages:

·        If any required field is blank, a message is displayed listing the required field(s) that are blank. Complete those fields, then click the Generate Local Product # button again.

·        If the Title ID is blank, a prompt (shown) is displayed, asking if you want Title ID to match the Local Product Number you’re generating. Do one of the following:

·        Click OK, and the local product number is entered into the Title ID field.

·        Click Cancel and a prompt reminds you to enter a Title ID. Click OK to continue.

Keep in mind…

When local product number is generated and the Config Code indicates the product is a single, the Local Product Number is entered automatically into the Title ID field as Legacy does not permit linking singles using Title ID.
