Selection Criteria

The selection criteria can vary for each report. Below is a list of all the selection criteria fields available for the reports.

Field Name

Description

Begin Street Date

The first street/ship date of those releases within a date range to be included in the report

Ending Street Date

The last street/ship date of those releases within a date range to be included in the report

Beginning Due Date

The first task due date for releases within a date range to be included in the report

Ending Due Date

The last task due date for releases within a date range to be included in the report

Begin Effective Date

The first date for releases for a custom date range. It is currently used to determine if records have been added or if street dates have changed.

Ending Effective Date

The last date for releases for a custom date range. It is currently used to determine if records have been added or if street dates have changed.

Distribution

Request to view all releases sharing the same UMVD distribution region

Release Type

Request to view all releases for a specified release type, commercial or promo

Status

Request to view all releases sharing the same status. The options are Closed, Active (active & TBS), Active excluding TBS, All (Closed, Cancelled, Active, & TBS), and TBS.

Label Contact

Request to view all releases for a particular label contact

UML Contact

Request to view all releases for a particular UML contact

Task Owner

Request to view only tasks owned by the only the label, only UML, or both.

Artist

Request to view all releases for a particualr artist

Schedule Releases

Request to view only releases with templates applied

Parents Only

Request to view only releases with only display parent records, records that have matching title ID and prefix/selection Ids

Family

Request to view all releases for a specified family

Company

Request to view all releases for a specified company

Label

Request to view all releases for a specified Label

Configuration

Request to view all releases for a specified family

 

The Family name will automatically default to that under which the user operates. If a user operates under more than one Family, the Family name automatically defaults to "All". The Company and Label names will also automatically default to "All".

When defining criteria, if Family name alone were entered as the selection criteria then release information for all artists for all companies under that family would result. Likewise, if a date range for street/ship date were entered without any other selection criteria, all releases within those defined street/ship dates would display on the report.

If a user wants to view only key tasks, click in the option button next to UML to view UML key tasks or Label to view label-specific key tasks.
